# My Pokemon

Pokémon FireRed–style RPG built with React, TypeScript, Vite, Tailwind, and Zustand.
**All map data is canonically extracted from the pokefirered disassembly at build time** — zero hand-authored coordinates.

## Run

```bash
npm install
npm run dev          # → http://localhost:3000
npm run test:run
npm run lint
```

If `npm` is not found on macOS:

```bash
export PATH="/opt/homebrew/bin:$PATH"
```

## Map pipeline (fully programmatic)

Three scripts run automatically via `predev`/`prebuild`. Edit `pokefirered_dissasembly/`, run `npm run dev`, the game reflects it on next paint.

| Script | Output |
| --- | --- |
| `scripts/build-firered-pipeline.mjs` | Extracts 365 layouts + 63 tilesets from `pokefirered_dissasembly/` to `src/artifacts/firered/`. |
| `scripts/stitch-firered-overworld.mjs` | BFS-stitches the 38 outdoor Kanto zones via FireRed's `connections` graph; emits `STITCHED_KANTO_OVERWORLD.json` + `kantoZoneOffsets.generated.ts`. |
| `scripts/generate-indoor-maps.mjs` | Auto-emits the 77-entry indoor map registry + FireRed↔internal id mapping. |

Re-run manually:

```bash
npm run generate:firered
```

### Renderer

`src/components/overworld/FireredMapView.tsx` is a canvas-based metatile renderer. At render time it composes each metatile from the canonical FireRed tile sheet + palettes, applying per-tile flips and the right palette of 16 colors (palette slots 0–6 from primary tileset, 7–12 from secondary, per `pokefirered/include/fieldmap.h`).

The unified Kanto overworld renders as multiple `FireredMapView` instances — one per zone — at offsets computed by the stitcher.

## Project structure

```
scripts/
  build-firered-pipeline.mjs     extract pokefirered → src/artifacts/firered/
  stitch-firered-overworld.mjs   compute Kanto offsets via connection graph
  generate-indoor-maps.mjs       emit indoor map registry + id mapping

src/lib/firered/
  bridge.ts                      single-layout → MapData
  multiZoneBridge.ts             stitched Kanto → MapData
  tilesetLoader.ts               runtime metatile bitmap composer

src/data/maps/index.ts           tiny registry, ~70 lines
src/data/firered/                autogenerated outputs

src/components/overworld/FireredMapView.tsx     canvas renderer (used everywhere)

src/lib/battleEngine.ts          pure battle state machine
src/lib/damage.ts                damage / stat / type math
src/store/gameStore.ts           Zustand store
src/test/simulator/              headless GameSimulator (integration tests)
```

## Controls

- Arrow keys — move
- `Enter` / `Z` — interact
- `X` / `Esc` — open menu
- `G` — ghost mode (walk through walls)

## Preview routes

- `/` — full game
- `/?firered=KANTO` — standalone preview of the entire stitched Kanto with FireRed art (no game state)
- `/?firered=LAYOUT_PALLET_TOWN_PROFESSOR_OAKS_LAB` — single-layout preview for any of the 365 FireRed maps

## Testing

```bash
npm run test:run   # CI
npm test           # watch
```

Notable tests:

- `src/data/__tests__/fireredPipeline.test.ts` — pipeline structural contract (every map has a grid + collision, every tileset has 16-color palettes + 8-tile-ref metatiles, etc.)
- `src/data/__tests__/warpRoundTrip.test.ts` — every indoor exit warp lands within 1 tile of the outdoor entry that sent the player in (catches the "middle of nowhere" bug class)
- `src/test/simulator/scenarios.test.ts` — story integration scenarios (Oak intercept, Mom heal, rival battle, wild encounters, gym leaders, etc.)
