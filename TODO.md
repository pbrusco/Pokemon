## Architecture notes (read before working)

- **All map data is auto-extracted** from `pokefirered_dissasembly/` via `npm run generate:firered` (runs automatically on `npm run dev`/`npm run build`). The disassembly is a git submodule — `git submodule update --init` after a plain clone. Generated artifacts under `src/artifacts/firered/{maps,tilesets}/` are gitignored and regenerated by the pipeline.
- Zone offsets live in `src/data/firered/kantoZoneOffsets.generated.ts` — never hand-edit. To shift a zone, fix it in the disassembly's `connections` data and re-run the pipeline.
- NPC data: `src/data/npcDatabase.ts` → `buildNPCDatabase()`. Coordinates use `w('ZONE_NAME', lx, ly)` against the auto-generated offsets. Most NPC/item/trainer data flows from `extract-game-data.mjs` and is merged in via `mergeAuto()`; only Pallet/Oak's-Lab story NPCs are hand-authored.
- The internal `MapID` ↔ FireRed mapping is in `scripts/generate-indoor-maps.mjs` (`MAP_ID_TO_FIRERED`). Two value forms: a string (1:1 layout↔map) or `{ layout, map }` for shared layouts (e.g. every city's Pokémon Center shares `LAYOUT_POKEMON_CENTER_1F`).
- Run `bash .githooks/pre-commit` before finishing any task (tsc + vitest + knip).
